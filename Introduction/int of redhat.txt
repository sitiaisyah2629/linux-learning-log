EXPLAIN DIR IN ROOT
/ in root, we have:
afs
bin
boot # (linux kernel needed to start the system live. dont touch unless doing troubleshooting boot problem)
dev
etc # (Extended Text Configuration. store all configuration files for the system. if changing how system works, the chances that im editting something below /etc) - do backup configuration before change them!!
home (personal)
lib
lib64
media
mnt (temporarily mounting storage like usb drive or network shares)
opt
proc
root (/root home directory for root user and work as super user. normal users cant access)
run (linux keeps all sorts of information what is happening right now on your system. start refresh every boot and wiped out when you start the system)
sbin
srv
sys
tmp # (temporary. just save here for temporary but not for the important file because all users in linux system has access in /tmp)
usr # (user program. most of the software are install are ends up here /usr/bin (bin - binaries represent executable programs)). libraries below /user/lib64. /usr/shared to find shared resources like documentation.
var # (variable stuff that change a lot such as logs, mail, print job)

Active Directory or Redhat Identity Directory Management (IDM)
userdel -r (username) # delete its all dir acc
userdel (username) # delete its acc
BEST PRACTICE IS TO NOT DELETE, INSTEAD JUST LOCK THE ACCOUNT

- every user have unique ID (UID)
- if delete the user, that user ID can be reused later
- if the new user get the same user ID, they could owning files that belong to the previous user. Accidently exposed the sensitive data

id (username) # to see the its UID
passwd -l (username) # lock the account
find / -uid 1001 2>/dev/null # find the directory or all find under 1001 UID. DELETE ALL OF THIS BEFORE YOU FULLY REMOVING THE USER ACCOUNT

add a local user to a group
usermod -aG wheel (username) # add the local user to a wheel group. wheel is the default administrator group on RHEL systems. anyone in this group can use the sudo command to run administrative task. Only add people that need to run administrator task in this group.

exit # to exit the root (back to a local user)
su - # subtitute user or switch user to the root user
sudo -i # run administrative task from local acc. all this audited.

ERRATA
(RHBA) Red Hat Bug Advisory - bug fixes
(RHEA) Red Hat Enhancement Advisory - improvemengt existing packages
(RHSA) Red Hat Security Advisory - self explanatory, security updates

- RHSA tied to (CVE) Common Vulnerabilities and Exposure
- CVE identified by IDs
- if packages affected by vulnerabilities, ill get CVE (28676) and CVSS score (9.8)

to check the packages that effected by errata:
dnf updateinfo list # list all errata
dnf updateinfo info RHSA -2025:10854 # check specific errata
dnf update # install to update entire system
dns update --security # update security
dns update --sec-severity=Importance --sec-severity=Critical # limit the updates to the highest priority security fixes

after update, please check either redhat need to boot or not
dns needs-restarting -r # if yes, type reboot. this will complete the packages update

MANAGING SYSTEM STARTUP SERVICES WITH SYSTEMD
systemd manages and many more: 
service - running application, deamon
socket - activating things on demand
timer -  for scheduling tasks
target - for grouping units
path - handle file n directories

dnf install -y httpd # install services
encho 'hello world' | tee /var/www/html/index.html # default document
systemctl start httpd # start the services. systemctl start httpd.service (can use this also) 
systemctl enable httpd # enable services
systemctl --now httpd # start and enable the unit httpd right now 
systemctl status httpd # to view the status of httpd
systemctl restart (httpd) # restart the service without full boot
^restart^stop # change the word restart to stop. service will stop running httpd
^stop^disable # change the work restart to stop. and the service httpd will not run automatically after booting.

allow firewall traffic
firewall-cmd --add-service hhtp # add firewall rule (not permanent)
!! --permanent # run the previous command and make it permanent
firewall-cmd --list-all # list all the firewall rule
